# Vendored Strudel CodeMirror extensions

This directory carries vendored copies of Strudel's CodeMirror autocomplete
and Ctrl-hover tooltip extensions, adapted to GXW's runtime conventions
(esm.sh imports, file-extension-explicit relative paths). The two
extensions back the "Enable auto-completion" and "Enable tooltips on Ctrl
and hover" preferences in GXW's settings dialog, matching the same toggles
in the Strudel REPL.

## Origin

Source: [Strudel monorepo](https://codeberg.org/uzu/strudel),
package `@strudel/codemirror`. Vendored from version 1.3.0.

## Files

| File | Origin | Notes |
| ---- | ------ | ----- |
| `autocomplete.mjs` | `@strudel/codemirror/autocomplete.mjs` | Imports rewritten for our esm.sh / relative-path conventions. `complex` from `@strudel/tonal` is shimmed to an empty object for the chord-symbol completion (chord handler still completes pitches; symbols are a follow-up). |
| `tooltip.mjs` | `@strudel/codemirror/tooltip.mjs` | Imports rewritten. Otherwise unchanged. |
| `html.mjs` | `@strudel/codemirror/html.mjs` | Unchanged. |
| `doc.js` | (GXW) | Merger module: imports `strudel-doc.js` and `gxw-doc.js`, exports a unified `{ docs: [...] }` object the vendored extensions read. GXW entries come last so they win on any name collision. |
| `strudel-doc.js` | auto-generated | The JSDoc data dump from the Strudel monorepo, slimmed to the fields the renderer reads. Regenerated by `scripts/refresh-strudel-doc.sh`. |
| `gxw-doc.js` | (GXW) | Hand-edited entries for GXW-specific functions, signals, helpers worth surfacing in the editor. Same schema as Strudel's. |

## Refreshing the Strudel data

Run from the repo root:

```
scripts/refresh-strudel-doc.sh
```

This clones the Strudel monorepo into a temp directory, installs JSDoc
and the jsdoc-json template, runs Strudel's documentation build step,
slims the output, and writes `strudel-doc.js`. The result is a single
file change; inspect the diff and commit when you're satisfied.

Requirements: `git`, `node`, `npm`, internet access. Strudel's package
manifest uses pnpm workspace protocol that plain `npm` can't resolve,
so the refresh script installs `jsdoc` and `jsdoc-json` into an isolated
working directory rather than running `npm install` against the Strudel
root.

## Adding GXW entries

Edit `gxw-doc.js` directly. The schema is documented at the top of that
file; each entry mirrors what JSDoc produces for a documented Strudel
function. Entries appear in the editor's autocomplete popup and (when
the user Ctrl-hovers a matching identifier) in the tooltip, alongside
the Strudel entries from `strudel-doc.js`.
